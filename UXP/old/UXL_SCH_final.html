UXL · 3. 검색 — 검색 화면의 고객 경험 원칙 정의

UXL_SCH — 검색

고객의 검색 의도를 정확히 이해하고 가장 적합한 결과를 즉시 제시해, 원하는 정보에 최단 경로로 도달하도록 설계합니다.

의도 분석 탐색 맥락 적합 결과 제시 검색 내 즉시 행동 유도 결과 신뢰성 및 투명성 확보

UX Principle Details

UXL_SCH_1

3-1. 의도 중심 검색 해석

키워드 검색 시 단순 문자열 매칭이 아닌 고객의 이용 맥락을 반영하고, 자연어 입력 시 의도가 복수로 해석될 수 있는 쿼리에는 후속 질문이나 추천 검색어를 자연스럽게 제안하여 검색 정확도를 높여야 합니다.

BEHAVIOR

고객은 키워드를 입력하고 여러 검색 결과를 탐색하며 원하는 정보를 직접 찾아야 합니다. 의도가 명확하지 않은 경우 여러 번 검색을 반복합니다.

AS-IS

검색이 순수 문자열 매칭으로만 동작하여 고객의 상황이나 맥락이 반영되지 않습니다. 자연어 입력이나 모호한 검색 의도는 정확히 해석되지 못합니다.

TO-BE

고객의 이용 맥락과 입력 의도를 함께 해석해 검색 유형에 맞는 탐색 방식으로 연결해야 합니다. 의도가 명확하지 않은 경우에는 후속 질문이나 추천 검색어를 통해 의도를 구체화해야 합니다.

pattern_id UXL_SCH_1
screen_type search_result
trigger search.input_text IS NOT NULL
output SearchResultPage → slot: search_primary, state: intent_resolved
input_data search_query (manual), search_type (natural_language | keyword), user_context (profile)
fallback search_type unresolvable → keyword_search_result as default
RULE 1
사람

검색 입력은 단순 키워드가 아니라 고객 의도를 기반으로 해석하고, 검색 유형에 따라 적절한 탐색 방식으로 연결해야 합니다.

AI
rule_id UXL_SCH_1_RULE_1
condition search_type == 'natural_language' OR search_type == 'keyword'
route_NL agent_conversation_flow (의도 해석 → 질문 기반 탐색)
route_KW integrated_search_result_page (personalized_ranking + 추천 검색어)
ambiguous search_intent IS ambiguous → followup_question OR recommend_search_terms
Do
사람

검색 유형을 구분해 고객 맥락에 맞는 탐색 방향을 제안합니다. 자연어 입력 시 에이전트 흐름으로, 키워드 입력 시 개인화 통합 결과로 연결합니다.

AI
if NL query → agent_conversation_flow (intent clarification)
if KW query → integrated_search_result + personalized_ranking
if ambiguous → render clarification_question + recommended_search_terms
AI
퍼플 블록에이전트 대화 흐름 — 자연어 의도 해석 및 질문 기반 탐색
라이트 블록개인화 통합 검색 결과 — 맥락 기반 우선순위 정렬
Don't
사람

검색 유형 구분 없이 동일한 방식으로 처리하지 않습니다. 자연어 쿼리에 단순 문자열 매칭을 적용하거나 의도가 모호한 경우 결과 없음 화면만 반환하지 않습니다.

AI
reject NL query에 keyword string-matching 적용 금지
reject ambiguous query → zero-result page without alternatives 금지
fallback → agent_conversation_flow / clarification_question 제공
AI
회색 블록단순 키워드 매칭 결과 — 개인 맥락·의도 미반영
단절 상태모호한 쿼리에 대한 후속 안내 없이 결과만 나열

Scenario. 검색 유형별 탐색 경로 연결

자연어 검색
사람

"기기 바꾸고 싶은데 최대 할인 받으려면?" 같은 자연어 입력 시, 에이전트 대화 흐름으로 연결해 질문 기반 탐색과 실행을 지원합니다.

AI
trigger search_type == 'natural_language'
route agent_conversation_flow
example "기기 바꾸고 싶은데 최대 할인 받으려면?"
output intent clarification questions → 맥락 기반 추천 실행
키워드 검색
사람

"아이폰 21" 같은 키워드 입력 시, 개인화 순위가 적용된 통합 검색 결과 화면으로 연결하되 추천 검색어를 함께 제시해 이탈을 방지합니다.

AI
trigger search_type == 'keyword'
route integrated_search_result_page
ranking personalized_ranking (user_context 기반)
supplement 추천 검색어 카드 + 관련 질문 제안

UX Principle Details

UXL_SCH_2

3-2. 맥락 적합 결과 제시

동일 키워드라도 고객마다 최적화된 결과와 순서를 제공하며, 검색 결과가 없거나 부족할 때도 대안 경로를 함께 제공해야 합니다.

BEHAVIOR

고객은 동일한 검색어에 대해 모든 결과를 확인하며 자신에게 맞는 정보를 직접 선별합니다. 결과가 없거나 부족한 경우 다시 검색하거나 이탈합니다.

AS-IS

검색 결과가 모든 고객에게 동일하게 제공되어 개인 상황에 맞지 않는 정보가 포함됩니다. 결과가 없을 경우 대안이 없어 탐색이 단절됩니다.

TO-BE

고객의 이용 이력과 맥락에 따라 결과와 순서를 개인화해야 합니다. 결과가 없거나 부족한 경우에도 대안 경로를 함께 제공해야 합니다.

pattern_id UXL_SCH_2
screen_type search_result
trigger search.query IS NOT NULL
output PersonalizedSearchResult → slot: search_results, state: personalized
input_data search_query, user_history (profile), user_context (profile)
fallback user_history unavailable → platform_ranked_result_list
RULE 1
사람

검색 결과는 고객의 상황과 맥락에 맞춰 개인화된 구조로 제공해야 합니다. 결과가 없을 때도 대안 경로를 함께 제시합니다.

AI
rule_id UXL_SCH_2_RULE_1
if result > 0 reorder result_list by user_context (usage_history + context)
if result == 0render alternative_path_card + suggested_search_terms
reject identical result order for all users 금지
Do
사람

고객의 이용 이력과 상태를 반영해 결과의 우선순위와 구성을 다르게 제공해야 합니다. 결과가 없는 경우 이어서 탐색할 수 있는 대안을 함께 제시합니다.

AI
sort result_position_1 = most_relevant_item (user_context 기반)
zero_state alternative_path_card 렌더링 (suggested next steps 포함)
copy "이어서 검색해보세요" + 추천 검색어 카드
AI
퍼플 블록user_context 기반 최우선 결과 — 개인화 순위 적용
라이트 블록이하 결과 목록 — 이용 이력 반영 정렬
Don't
사람

모든 고객에게 동일한 결과와 순서를 일괄적으로 노출하지 않습니다. 결과가 없을 경우 대안 없이 빈 화면만 반환하지 않습니다.

AI
reject platform_ranking 고정 순서 일괄 노출 금지
reject result_count == 0 → empty state without guidance 금지
AI
회색 블록모든 고객 동일 순서 — 개인화 미적용
단절 상태결과 없음 시 대안 경로 미제공

Scenario. 결과 유무별 처리 방식

결과 있음 + 개인화 적용
사람

이용 이력과 현재 상태를 기반으로 결과 우선순위를 재구성합니다. 같은 키워드라도 고객마다 상이한 결과 순서를 경험합니다.

AI
trigger user_history IS NOT NULL AND result_count > 0
action result_list reorder by user.usage_history + user.context
position_1most_relevant_item (personalized score 최상위)
결과 없음 — 대안 경로
사람

결과가 없을 때 빈 화면 대신 "이어서 검색해보세요"와 추천 검색어 카드를 제공해 탐색이 단절되지 않도록 합니다.

AI
trigger result_count == 0
component alternative_path_card
copy "이어서 검색해보세요" + 추천 검색어 카드

UX Principle Details

UXL_SCH_3

3-3. 맥락 내 즉시 행동 유도

검색 결과를 단순 나열에 그치지 않고 각 항목에서 바로 실행 가능한 액션을 함께 제공하여, 검색에서 태스크 완료까지의 단계를 최소화해야 합니다.

BEHAVIOR

고객은 검색 결과를 확인한 뒤 상세 페이지로 이동해 추가 탐색을 수행한 후 행동합니다. 여러 단계를 거치며 이탈이 발생합니다.

AS-IS

검색 결과가 단순 정보 나열에 그쳐 행동까지 이어지지 않습니다. 고객은 결과 확인 이후 별도의 경로를 통해 다시 이동해야 합니다.

TO-BE

탐색과 실행 단계를 통합해 검색 결과에서 바로 실행 가능한 액션을 제공해야 합니다. 고객은 추가 탐색 없이 결과에서 바로 행동으로 이어질 수 있어야 합니다.

pattern_id UXL_SCH_3
screen_type search_result
trigger search_result.item_count > 0
output SearchResultItem → slot: search_results, condition: item.action_available == true, state: action_embedded
input_data search_result_items (real_time_api), user_context (profile)
fallback action_available == false → render result_item without inline action
RULE 1
사람

검색 결과 내에서 탐색과 실행이 분리되지 않도록 즉시 행동 가능한 구조를 제공해야 합니다.

AI
rule_id UXL_SCH_3_RULE_1
if action_availableembed inline_action_button within result_item (no page nav)
if terminate show termination_method_selector inline (즉시/예약 선택)
if subscribe show subscription_cta inline + contextual copy
reject 별도 action screen redirect 금지
Do
사람

각 결과 항목에서 바로 실행 가능한 행동을 함께 제공해 추가 탐색 없이 완료까지 이어지도록 합니다. 결과 영역 내에서 즉시 실행할 수 있도록 구성합니다.

AI
embed inline_action_button (result_item 내부 포함)
sub_copy "현재 {current_product}에 매달 {current_price}원 결제 중이에요."
offer_copy "동일한 금액에 {benefit_desc}까지 포함된 구독 상품이 있어요."
AI
퍼플 블록인라인 액션 영역 — 별도 화면 이동 없이 결과 내 즉시 실행
하단 버튼CTA — 해지/가입 등 행동 유도 버튼 result_item 내 포함
Don't
사람

결과 확인 이후 별도의 화면 이동이나 추가 탐색을 요구하지 않습니다. 행동이 필요한 경우에도 결과 영역 내에서 즉시 처리될 수 있어야 합니다.

AI
reject action required → separate action screen redirect 금지
reject 결과 나열 후 행동 진입 경로 미제공 금지
AI
회색 블록결과 목록만 나열 — 인라인 액션 미포함
단절 화살표별도 상세 페이지 이동 필요 → 이탈 위험

Scenario. 결과 내 인라인 액션 유형

구독 해지 검색 결과
사람

구독 해지 관련 검색 결과에서, 별도 화면 이동 없이 즉시해지·예약해지 선택을 인라인으로 제공합니다.

AI
trigger result_action == 'terminate_subscription'
component termination_method_selector (즉시해지 | 예약해지) inline
copy "즉시해지는 지금 바로 종료되고, 예약해지는 이용 기간이 끝나는 날 해지돼요."
구독 상품 검색 결과
사람

구독 상품 검색 결과에서, 가입 CTA와 동일 금액 대비 혜택 비교를 인라인으로 제공해 탐색 없이 바로 결정할 수 있게 합니다.

AI
trigger result_action == 'subscribe'
component subscription_cta inline + benefit_comparison
copy "현재 {current_product}에 매달 {current_price}원 결제 중이에요. 동일한 금액에 {benefit_desc}까지 포함된 구독 상품이 있어요."

UX Principle Details

UXL_SCH_4

3-4. 결과 신뢰성 및 투명성 확보

AI 기반 개인화 검색 결과에 대해 고객이 왜 이 결과가 제시되었는지 이해할 수 있도록 근거를 간결하게 노출하고, 결과에 대한 신뢰를 형성해야 합니다.

BEHAVIOR

고객은 검색 결과가 왜 노출되었는지 이해하지 못한 채 결과를 수용하거나 의심합니다. 특히 개인화 결과에 대한 신뢰가 낮습니다.

AS-IS

개인화 근거가 부족해 고객이 결과를 신뢰하기 어렵습니다. 이는 의사결정의 지연으로 이어집니다.

TO-BE

결과가 도출된 이유를 간결하게 제공해 고객이 납득 가능한 구조를 만들어야 합니다. 결과에 대한 신뢰를 기반으로 행동이 이어지도록 설계합니다.

pattern_id UXL_SCH_4
screen_type search_result
trigger search_result.personalization_applied == true
output PersonalizationRationale → slot: search_result_meta, state: visible
input_data personalization_reason (real_time_api)
fallback personalization_reason unavailable → render result without rationale label
RULE 1
사람

검색 결과는 고객이 납득할 수 있는 근거와 함께 제공되어야 합니다. 결과가 도출된 이유를 고객의 맥락과 연결해 간결하게 설명합니다.

AI
rule_id UXL_SCH_4_RULE_1
if personalized render brief rationale_label (user_context 연결)
if AI_recommend render reasoning_snippet (usage_history | contract_status 기반)
reject "recommended for you" 단독 레이블 금지 (구체적 맥락 명시 필수)
copy_template "{user.contract_years}년 장기 고객 기준 최대 {max_discount}원 특별 혜택이 있어요"
Do
사람

결과가 제시된 이유를 고객의 맥락과 연결해 간결하게 설명합니다. "장기 고객 기준", "이용 이력 기반" 등 구체적 근거를 레이블로 표시합니다.

AI
rationale PersonalizationRationale 컴포넌트 → result 상단 레이블
context_linkuser_context (contract_years, usage_history) → 레이블 텍스트에 반영
example "SKT 추가 혜택 (장기 고객 {N}년)"
AI
상단 레이블PersonalizationRationale — 고객 맥락 기반 추천 근거 표시
체크 아이콘신뢰 형성 — 고객이 납득할 수 있는 구체적 이유 제공
Don't
사람

근거 없이 결과만 제시하거나 판단 이유를 숨기지 않습니다. "추천" 레이블만으로 고객이 직접 이유를 추정하게 하지 않습니다.

AI
reject personalized result에 근거 레이블 미노출 금지
reject opaque "추천" 레이블만 표시 금지 (구체 맥락 명시 필수)
AI
회색 블록결과만 나열 — 개인화 근거 미표시
불투명 상태고객이 왜 이 결과가 노출됐는지 알 수 없음

장기 고객 혜택 근거
사람

AI
