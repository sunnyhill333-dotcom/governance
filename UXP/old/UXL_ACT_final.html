UXL · 5. 실행/구매 — 실행·구매 화면의 고객 경험 원칙 정의

UXL_ACT — 실행/구매

보유한 고객 데이터를 유기적으로 호출해 입력과 선택 단계를 정보 확인의 과정으로 전환하고, 발생 가능한 리스크를 선제적으로 감지해 고객이 심리적 부담 없이 실행을 완료할 수 있도록 설계합니다.

입력/선택 간소화 선제적 리스크 처리

UX Principle Details

UXL_ACT_1

5-1. 입력/선택 간소화

보유한 고객 데이터를 유기적으로 호출해 입력과 선택 단계를 정보 확인의 과정으로 전환함으로써, 사용자의 물리적인 수고를 제거하는 '제로 입력' 여정을 구현해야 합니다.

BEHAVIOR

고객은 가입, 변경, 구매 과정에서 동일한 정보를 반복 입력하고 각 단계별로 옵션을 직접 선택합니다. 입력 오류와 재확인 과정이 반복되며 진행이 지연되거나 중단됩니다.

AS-IS

고객 데이터를 활용하지 않아 입력과 선택 과정이 유지되고 있습니다. 이로 인해 물리적 피로와 이탈 가능성이 증가합니다.

TO-BE

고객 데이터를 자동 호출하여 입력과 선택 단계를 축소하고, 고객이 정보 확인만으로 진행할 수 있는 구조로 전환해야 합니다. 입력은 선택이 아닌 시스템이 선제 수행해야 하는 영역이 되어야 합니다.

UX Principle Details

UXL_ACT_2

5-2. 선제적 리스크 처리

발생할 수 있는 리스크와 유의 사항을 시스템이 선제적으로 감지해 명확히 안내함으로써, 고객이 심리적 부담 없이 과업을 처리할 수 있도록 설계해야 합니다.

BEHAVIOR

고객은 위약금, 조건 변경, 불이익 등의 리스크를 사후에 인지하거나 직접 확인하려고 합니다. 불확실성이 존재할 경우 실행을 보류합니다.

AS-IS

리스크 정보가 분산되어 있거나 사전에 안내되지 않아 고객이 심리적 불안을 느끼고 의사결정을 미루게 됩니다.

TO-BE

시스템이 리스크를 사전에 감지하고, 명확한 영향 범위를 선제적으로 안내해야 합니다. 고객이 불안 없이 즉시 실행할 수 있는 상태를 만들어야 합니다.

pattern_id UXL_ACT_2
screen_type checkout_step
trigger action.risk_detected IS NOT NULL
output RiskAlertCard → slot: checkout_pre_action, condition: risk_detected == true / ProcessProgressIndicator → slot: checkout_status, condition: multi_step_process == true
input_data action_type (session), penalty_calculation, refund_calculation, benefit_impact, process_stage_list (real_time_api)
fallback risk_calculation unavailable → render confirmation without breakdown; append 'calculating...' label
RULE 1
사람

실행 이전 단계에서 리스크를 감지해 사전에 안내해야 합니다. 리스크를 숨기거나 실행 이후에 인지되도록 방치하지 않습니다.
