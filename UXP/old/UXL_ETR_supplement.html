UX Principle Details

UXL_ETR_1

1-1. 상태 기반 선제적 대응

진입 시점의 고객 데이터를 기반으로 상태를 진단해, 당장 확인이 필요한 알림이나 개인화된 맞춤 태스크를 최우선으로 노출해야 합니다.

BEHAVIOR

고객은 앱 진입 시 전체 메뉴를 탐색하기보다 당장 필요한 기능이나 상태 확인부터 시도합니다.

AS-IS

진입 시 일반화된 홈 구조로 인해 고객은 자신의 상황과 무관한 정보 속에서 필요한 행동을 다시 찾아야 합니다.

TO-BE

C360 및 행동/의도 데이터 기반으로 현재 상황을 즉시 해석하고, 고객별로 다른 우선순위의 마이크로 태스크를 선제적으로 노출해야 합니다.

pattern_id UXL_ETR_1
screen_type home_entry
trigger session.start == true AND customer_data.available == true
output_component ProactiveMicroTaskCard → slot: home_primary, max_count: 1
input_data customer_status, pending_event, recent_action_trigger
suppressed_when customer_data.available == false → fallback: static_home_layout
SUPPLEMENT 1

복수 시나리오의 trigger가 동시에 만족될 경우, 아래 우선순위 규칙으로 단일 카드를 선택합니다. max_count: 1 제약과 함께 적용됩니다.

conflict_resolution highest-priority-wins — 동시 충족 시 priority 값이 낮은 시나리오 1개만 렌더링
priority_order 1. family_data_request.pending → 2. contract.days_until_expiry ≤ 7 → 3. payment.status == pending
tie_break 동일 priority 충돌 시 → pending_since 최신순 (가장 최근 발생 이벤트 우선)
re_evaluate_on session.resume OR customer_data.refresh (재진입·데이터 갱신 시 재평가)
