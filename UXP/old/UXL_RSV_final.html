UXL · 6. 문제해결/CS — CS 화면의 고객 경험 원칙 정의

UXL_RSV — 문제해결/CS

고객이 문제를 체감하거나 문의하기 전에 이상 징후를 감지해 선제적으로 안내하고, 채널 간 상담 맥락을 단절 없이 연결하며, 발화 의도를 실시간으로 해석해 즉시 실행 가능한 해결 흐름을 완성합니다.

선제적 문제 감지 단절 없는 상담 맥락 연결 인텐트 기반 실시간 상담 가이드

UX Principle Details

UXL_RSV_1

7-1. 선제적 문제 감지

고객이 문제를 체감하거나 직접 문의하기 전에 이상 징후를 감지하여, 문제 상황을 사전에 안내하고 즉각적인 해결 경로를 제공해야 합니다.

BEHAVIOR

고객은 문제가 발생한 이후에야 원인과 해결 방법을 찾기 위해 문의나 탐색을 시작합니다.

AS-IS

이상 징후가 사전에 포착되지 않아 고객이 문제를 인지한 뒤에야 대응이 시작되며, 이 과정에서 불필요한 추가 행동이 발생합니다.

TO-BE

고객의 이용 패턴과 상태 데이터를 기반으로 문제를 선제적으로 감지하고, 고객이 인지하기 전에 해결 경로를 제시해야 합니다.

pattern_id UXL_RSV_1
screen_type cs_resolution
trigger user.anomaly_signal IS NOT NULL
output ProactiveAlertCard → slot: home_or_cs_entry, condition: anomaly_signal IS NOT NULL, state: proactive_visible
input_data usage_anomaly_data (real_time_api), billing_anomaly_data (real_time_api), benefit_utilization_data (profile)
fallback anomaly_data unavailable → render standard cs_entry without proactive alert
RULE 1
사람

고객의 문제 해결에 필요한 정보와 대응 경로를 선제적으로 제공해야 합니다. 고객이 직접 문제를 발견하고 해결책을 탐색할 때까지 기다리지 않습니다.

AI
rule_id UXL_RSV_1_RULE_1
if data_warning render DataWarningCard + remaining_data + gift_data_cta (exhaustion < 24h)
if billing_anomaly render BillingAnomalyAlert + anomaly_reason + resolution_cta
if unused_benefit render UnusedBenefitAlert + benefit_count + usage_guide_cta
proactive surface resolution_path before user_inquiry (not after)
reject wait for user CS contact before surfacing issue 금지
Do
사람

고객 데이터를 분석하여 이상 징후를 감지하고, 해결 방안까지 선제적으로 제공합니다. 홈 진입 시 상태 분석 결과를 기반으로 맞춤 태스크를 안내합니다.

AI
render ProactiveAlertCard at home_entry or cs_entry
include anomaly_reason + resolution_cta (즉시 해결 경로)
copy "지금 사용 패턴이라면, 어머니의 데이터는 내일 모두 소진될 가능성이 있어요."
AI

화살표resolution_path — 고객 문의 없이 선제 해결 경로 연결

사람

reject resolution_path provided only after user_inquiry 금지
AI

단절 상태해결 경로 미제공 → 불필요한 CS 문의 발생

UX Principle Details

UXL_RSV_2

7-2. 단절 없는 상담 맥락 연결

온·오프라인 채널의 상담 이력과 진행 상태를 실시간으로 동기화하여 접점 이동 시에도 고객이 반복 설명하지 않도록, 단절 없는 옴니채널 경험을 제공해야 합니다.

BEHAVIOR

고객은 문제 해결 과정에서 앱, 상담원, 오프라인 등 다양한 채널을 이동할 때마다 동일한 상황과 이력을 반복 설명하며 상담을 이어갑니다.

AS-IS

채널 간 데이터와 진행 상태가 연결되지 않아, 고객이 채널을 이동할 때마다 동일한 문제를 다시 설명하고 해결 과정을 반복해야 합니다.

TO-BE

모든 채널에서 고객의 상태와 상담 맥락이 하나의 흐름으로 연결되어, 접점이 바뀌어도 동일한 문제 해결 과정이 끊김 없이 이어지도록 해야 합니다.

pattern_id UXL_RSV_2
screen_type cs_resolution
trigger consultation.channel_switch == true
output ContinuedConsultationView → slot: cs_primary, state: context_preserved
input_data consultation_history (session), task_progress_stage (session), channel_target: app | agent | offline_store
fallback consultation_history unavailable → render fresh cs_entry with apology for context loss
RULE 1
사람

고객의 상담 여정은 채널과 관계없이 단일 여정으로 유지되어야 하며, 이전 상태를 기반으로 즉시 이어질 수 있어야 합니다. 이전 맥락을 재확인하거나 추가 입력을 요구하지 않습니다.

AI
rule_id UXL_RSV_2_RULE_1
if channel_switch restore consultation_history + task_progress_stage in new channel (no re-entry)
if offline_transfer transfer task_progress_stage to nearest_store + surface 'IMEI 입력 단계부터' label
smart_planner 스마트플래너 상 데이터 연동 — 진행 단계 동기화
reject reset task_progress or clear consultation_history on channel switch 금지
Do
사람

채널 전환 시 고객의 현재 상태와 진행 단계가 다음 채널에서 즉시 이어지도록 구성합니다. 태스크 진행 중 어려움이 감지되면 오프라인 매장 연결까지 지원합니다.

AI
preserve consultation_history + task_progress_stage → destination channel
offline "IMEI 입력 단계부터 개통 진행" 레이블 매장에 표시
sync 스마트플래너 상 데이터 연동
AI
채널 아이콘앱 → 매장 전환 — 진행 단계 실시간 동기화
연결 화살표단일 여정 유지 — 맥락 초기화 없이 이어짐
Don't
사람

이전 맥락을 재확인하거나 추가 입력을 요구하여 고객이 다시 시작하도록 만들지 않습니다.

AI
reject task_progress_stage reset on channel switch 금지
reject user re-explain situation or re-enter data 요구 금지
AI

단절 상태채널 이동 시 맥락 손실 → 반복 설명 유발

매장 IMEI 입력 안내

UX Principle Details

UXL_RSV_3

7-3. 인텐트 기반 실시간 상담 가이드

고객의 현재 상태와 상담 맥락(발화, 인텐트)을 실시간으로 분석하여 상담에 필요한 메뉴와 대응 정보를 즉시 제공하고, 처리 시 후속 업무까지 선제적으로 연결하여 재문의 없는 해결 방안을 제공해야 합니다.

BEHAVIOR

고객은 상담 중 필요한 정보를 확인한 뒤 다음 행동을 스스로 판단하고 별도의 단계로 이동해 문제를 해결합니다.

AS-IS

상담 과정에서 정보 제공과 실행이 분리되어 있어 고객이 다음 행동을 결정해야 하며, 이로 인해 해결까지의 단계가 늘어나고 중간 이탈이 발생합니다.

TO-BE

고객의 발화 의도와 현재 상태를 실시간으로 해석하여 필요한 정보와 실행 태스크를 함께 제시하고, 후속 조치까지 연결해야 합니다. 정보 제공에 그치지 않고, 즉시 실행까지 이어지는 해결 흐름을 완성해야 합니다.

pattern_id UXL_RSV_3
screen_type cs_resolution
trigger consultation.user_utterance IS NOT NULL
output IntentResolvedActionCard → slot: conversation_inline, condition: intent_resolved == true, state: action_ready
input_data user_utterance (manual), user_current_state (real_time_api), consultation_context (session)
fallback intent_resolution failed → render open_question_prompt asking user to clarify intent
RULE 1
사람

고객의 발화 의도와 상태를 실시간으로 해석하여, 필요한 정보와 실행 태스크를 후속 단계까지 끊김 없이 연결해야 합니다. 대화 중 별도 탐색을 유도하거나 상담 흐름이 끊기지 않도록 합니다.

AI

reject redirect to separate task screen outside conversation 금지
Do
사람

상담 중 고객의 요청이 대화 안에서 바로 해결되도록 구성합니다. 즉시 처리까지 대화 흐름 내에서 완결됩니다.

AI

refund pre-computed breakdown: 총구독가 - 이용기간차감 - 쿠폰차감 = 최종환불
copy "즉시해지를 선택하면 오늘 구독이 바로 종료돼요. 환불 예상 금액을 먼저 확인해 주세요."
AI

Don't
사람

대화 중 별도 탐색을 유도하여 상담 흐름이 끊기거나, 고객이 직접 다음 행동을 찾도록 만들지 않습니다.
