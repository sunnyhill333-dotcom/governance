UX Principle Details

UXL_ETR_1

1-1. 상태 기반 선제적 대응

진입 시점의 고객 데이터를 기반으로 상태를 진단해, 당장 확인이 필요한 알림이나 개인화된 맞춤 태스크를 최우선으로 노출해야 합니다.

BEHAVIOR

고객은 앱 진입 시 전체 메뉴를 탐색하기보다 당장 필요한 기능이나 상태 확인부터 시도합니다.

AS-IS

진입 시 일반화된 홈 구조로 인해 고객은 자신의 상황과 무관한 정보 속에서 필요한 행동을 다시 찾아야 합니다.

TO-BE

C360 및 행동/의도 데이터 기반으로 현재 상황(이벤트, 사용 상태, 행동 트리거)을 즉시 해석하고, 고객별로 다른 우선순위의 마이크로 태스크를 선제적으로 노출해야 합니다. 이는 '고객의 의도를 기반으로 UX를 동적으로 재구성'하는 구조로 이어져야 합니다.

pattern_id UXL_ETR_1
screen_type home_entry
trigger session.start AND customer_data.available == true
output_componentProactiveMicroTaskCard → slot: home_primary
input_data customer_status, pending_event, recent_action_trigger
suppressed_whencustomer_data.available == false → fallback: static_home_layout

UX Principle Details

UXL_ETR_2

1-2. 끊김 없는 여정 재개

이전에 완료하지 못했던 여정(구매, 개통, 혜택 활성화 등)이 있다면, 진입 첫 화면에서 해당 지점으로 바로 이어갈 수 있도록 단일 재개 CTA를 제공해야 합니다.

BEHAVIOR

고객은 구매, 개통, 혜택 활성화 등 중단된 작업을 다시 찾기 위해 여러 화면을 반복 탐색합니다.

AS-IS

이전 행동 맥락이 유지되지 않아 고객이 어디까지 진행했는지 기억에 의존해야 하며, 이탈 가능성이 높아집니다.

TO-BE

직전 행동 데이터와 태스크 단계(탐색-비교-결정-실행)를 기반으로 중단 지점을 정확히 복원하고, 진입 즉시 이어서 실행할 수 있는 단일 경로를 제공해야 합니다. 이는 고객의 흐름을 끊지 않는 Intent 기반 Action 설계로 연결되어야 합니다.

pattern_id UXL_ETR_2
screen_type home_entry
trigger session.start AND incomplete_task.exists == true
output_componentResumeBanner → slot: home_top (홈 진입 첫 화면 상단 고정)
input_data incomplete_task.type, incomplete_task.step, saved_form_data
behavior_rule 입력/선택 정보 유지 상태로 중단 단계에서 즉시 재개 — 초기화 금지
suppressed_whenincomplete_task.exists == false → ResumeBanner 미표시

UX Principle Details

UXL_ETR_3

1-3. 구조적 명확성

진입 후 어느 탭으로 전환하더라도 현재 위치와 맥락을 명확히 인지할 수 있도록, GNB 구조와 화면 위계를 일관되게 유지해야 합니다.

BEHAVIOR

고객은 탭 이동이나 기능 전환 시 현재 위치와 맥락을 다시 해석하려고 시도합니다.

AS-IS

구조와 네비게이션이 일관되지 않으면 고객은 매 화면마다 새롭게 이해해야 하며, 인지 부담이 누적됩니다.

TO-BE

자산 유형이나 기능이 달라도 동일한 구조 원리를 유지하고, 현재 위치·상태·맥락을 항상 명확하게 드러내야 합니다. 화면마다 해석 없이 위치를 즉시 인지할 수 있는 Frictionless Navigation을 제공해야 합니다.

pattern_id UXL_ETR_3
screen_type any_tab (GNB_홈 / GNB_혜택 / GNB_쇼핑 / GNB_나의T)
trigger tab_navigation OR feature_transition
output_componentGNBLayout → 고정 구조: 타이틀 영역 / 콘텐츠 영역 / 퀵메뉴
layout_rule 모든 탭은 동일한 위계(타이틀→콘텐츠→퀵메뉴)를 유지 — 탭별 커스텀 구조 금지
state_visible 현재 활성 탭 명시 필수 / 이전 컨텍스트 소실 금지
