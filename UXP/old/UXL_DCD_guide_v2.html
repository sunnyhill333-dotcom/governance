UXL_DCD — 결정

고객의 요금제 사용량·데이터 패턴·단말 상태를 종합 분석해 최적안을 선제적으로 제안하고, 고객이 즉시 판단할 수 있는 근거 기반 비교 화면을 제공합니다.
이 문서는 AI 화면 생성의 유일한 기준(Single Source of Truth)입니다. 모든 규칙과 카피 문구는 기계가 해석할 수 있도록 명확하게 작성되어야 합니다.

고객 행동

고객은 요금제·혜택·단말 상태를 각각 확인하며 최적안을 스스로 찾기 위해 여러 화면을 반복 탐색합니다.

현재 문제

데이터가 존재함에도 종합 해석이 없어 개인화 제안이 부재합니다. 고객이 직접 계산해야 하는 부담이 생깁니다.

목표 상태

사용 패턴·상황·데이터를 입체 해석해 단일 최적안을 선제 제시합니다. 절감액과 근거를 함께 보여 즉시 판단 가능하게 합니다.

✓ 해야 합니다

  • 위약금 없는 경우 해당 사실을 명확히 표기합니다.
✗ 하지 않습니다
  • 근거 없이 추천만 표시하지 않습니다. saving_amount + usage_fit 필드를 반드시 포함합니다.
  • "좋은 혜택", "다양한 혜택" 같은 추상 표현만 사용하지 않습니다. 구체적 금액 또는 일상 용어로 변환해서 표시합니다.

근거 기반 비교 제공

decision_compare
배경
고객 행동

고객은 요금제와 상품이 나열된 상태에서 각 항목을 직접 계산하고 기억하며 판단합니다. 반복 탐색이 발생합니다.

현재 문제

동일한 나열 구조로 고객이 비교 기준을 스스로 만들어야 합니다. 현재 대비 변화를 파악하기 어려워 결정이 지연됩니다.

목표 상태

현재 계약 기준으로 변화되는 핵심 요소만 대조합니다. 고객은 '무엇이 어떻게 달라지는지'만 보고 즉시 결정합니다.

상황화면 처리

데이터 제공량이 늘어나는 경우 "데이터 +{N}GB" 형식으로 표시합니다. → DeltaLabel (data_delta)
월 비용이 줄어드는 경우 "월 -{N}원 절약" 형식으로 표시합니다. → DeltaLabel (cost_delta)
조건이 복잡한 비교 상황 핵심 변화 요약 카드를 전체 비교 화면 위에 추가합니다. → KeyChangeSummaryCard + ComparisonView
✓ 해야 합니다
  • 현재 대비 변화값(delta)을 중심으로 즉시 이해 가능하게 표현합니다.
  • 복잡한 조건 비교 시 변경되는 핵심 정보만 요약한 구조를 병행 제공합니다.
✗ 하지 않습니다
  • 절대값만 나열해서 고객이 직접 계산하게 만들지 않습니다. delta_view + change_direction 레이블을 렌더링합니다.
규칙 2
비교 화면 초기에는 의사결정 핵심 정보 3가지만 보여줍니다. 세부 조건은 숨겼다가 요청 시 펼칩니다.
이런 상황이라면 → 이렇게 처리합니다
상황화면 처리
표시할 비교 항목이 많은 경우 비용 변화 · 데이터 변화 · 혜택 변화 3가지만 기본 노출합니다. → 약관, 부가 조건은 '더 보기' 뒤로 숨김
고객이 상세 내용을 요청하는 경우 고객 액션(탭/버튼) 이후에만 상세 항목을 펼칩니다. → 기본값은 접힘 상태
✓ 해야 합니다
  • 핵심 3가지(비용/데이터/혜택 변화)만 기본으로 표시합니다.
✗ 하지 않습니다
  • 초기 화면에서 모든 세부 조건을 한꺼번에 펼쳐 보여주지 않습니다. 핵심 3종만 선별 표시하고, 나머지는 '더 보기' 처리합니다.
규칙 3
고객 유형(세그먼트)에 따라 비교 방식과 첫 번째 메시지를 다르게 구성합니다. 모든 유형에 동일한 화면을 적용하지 않습니다.

분석형 절감 금액과 이유 먼저 제시 "왜 요금제 변경이 필요한지 알려드릴게요."

시니어 데이터 증가량 한 가지만 단순하게 "데이터가 ${N}GB 늘어 영상통화도 더 여유롭게 이용하실 수 있어요."

✓ 해야 합니다
  • user_segment별로 비교 방식과 리드 문구를 다르게 구성합니다.
✗ 하지 않습니다
  • 모든 고객에게 동일한 비교 화면을 그대로 적용하지 않습니다. 세그먼트에 맞는 ComparisonView 변형을 적용합니다.
