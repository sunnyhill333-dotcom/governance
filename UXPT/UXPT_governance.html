글로벌 내비게이션
전체 서비스 최상위 내비게이션은 어느 화면에서든 일관된 구조로 고정 노출합니다.
Fundamental
Rules
R1글로벌 내비게이션은 모든 최상위 화면에서 동일한 위치·구조로 고정 노출합니다.
R2현재 활성 탭은 색상·굵기·인디케이터로 명확히 구분하여 위치 인지를 지원합니다.
R3내비게이션 아이템 레이블은 목적지를 명확히 전달하는 명사형 단어를 사용합니다.
R4하위 페이지 진입 시 글로벌 내비게이션은 숨기되, 복귀 경로(Back)를 명시적으로 제공합니다.
DO

활성 탭을 색상 + 인디케이터로 명확히 구분하고, 탭 레이블은 목적 중심 명사(홈·혜택·마이)로 작성합니다.

DON'T

활성/비활성 구분 없이 동일 스타일을 적용하거나, 레이블에 동사·문장형("혜택 보러가기")을 사용하지 않습니다.

홈-탭 연속 스크롤 구조
홈(AI 레이어)과 콘텐츠 탭(혜택·상품·마이)은 병렬 탭이 아닌 수직으로 연결된 단일 스크롤 공간입니다. 고객은 스크롤 다운으로 탭 영역에 진입하고, 스크롤 업으로 홈으로 복귀합니다.
New
Rules
R1홈 AI 레이어와 콘텐츠 탭은 수평 탭 전환이 아닌 수직 연속 스크롤로 연결됩니다. 홈은 별도 탭이 아닌 탭 영역 위의 고정 레이어입니다.
R2고객이 스크롤 다운하여 콘텐츠 탭 영역에 진입하면 탭바(혜택·상품·마이)가 화면 상단에 sticky 고정됩니다. 스크롤 위치가 탭 영역 임계값 이상일 때만 활성화됩니다. 단, NAV_4(홈-탭 수직 연속 스크롤)가 적용된 화면에서는 Zone A 스크롤 구간에 탭바를 sticky하지 않으며, Zone B 진입 임계점에서만 sticky 전환한다.
R3탭 영역 내 스크롤 최상단에서 추가 스크롤 업(overscroll) 시 홈 AI 레이어로 자연스럽게 복귀합니다. 별도 버튼 없이 제스처로 복귀 가능합니다.
R4탭 간 전환(혜택↔상품↔마이)은 좌우 스와이프 또는 탭바 탭으로 처리합니다 (NAV_2 적용). 탭 전환 시 각 탭의 스크롤 위치는 독립적으로 유지됩니다.
R5Deep task 모드(LLM 대화·태스크 실행) 진입 시 수직 스크롤 구조는 비활성화되고 태스크 전용 화면이 오버레이됩니다. 태스크 완료·취소 후 진입 전 위치로 복귀합니다 (SDUI_3 R4 참조).

New
Rules

BSS(C360+CVM)가 고객의 현재 상태와 긴급도를 분석해 렌더링하는 행동 유도 카드 단위입니다. "지금 이 고객에게 가장 필요한 단일 행동"을 최소한의 정보로 제시하고, 고객이 즉각 실행하거나 다음 여정으로 이어지도록 설계합니다. 홈 화면 상단의 핵심 UI 단위이며, ISR 신호 체계의 최종 출력 형태입니다.

필수태스크 레이블 — "지금 해야 할 것"을 한 문장 이내 명사+동사형으로 표기. (예: "데이터 선물 도착")
필수긴급도 신호 — 배지 또는 타임스탬프로 긴급도를 시각화. High: 색상 강조 배지 / Medium: 서브 텍스트 날짜.
필수CTA — 즉시 실행 가능한 버튼 1개. 라벨은 동사형 단문. (예: "지금 받기", "바로 변경", "1GB 전송하기")
선택컨텍스트 요약 — CTA 실행 결과를 1줄로 미리 안내. FRL 불안 해소와 통합 가능. (예: "위약금 없이 변경 가능해요")
선택타이머 — 포인트 소멸, 혜택 만료 등 시간 제약이 있을 때만 노출.
DO

Micro Task는 "지금 바로 할 수 있는 것" 하나만 담습니다. CTA 1개, 레이블 1줄, 컨텍스트 1줄. 고객이 카드를 읽지 않아도 CTA만 보고 실행할 수 있어야 합니다.

DON'T

Micro Task에 탐색형 링크, 다중 CTA, 긴 설명을 넣지 않습니다. 그것은 일반 콘텐츠 카드입니다. "더보기"는 Micro Task에 존재하지 않습니다.

노출 기준 & 스태킹 규칙
BSS가 산출한 Micro Task를 홈 화면에 몇 개까지, 어떤 순서로 노출할지 정의합니다.
New
Rules
R1Zone A 기본 상태에서는 SDUI_4 R2에 따라 1개만 노출한다. 다음 조건을 모두 충족하는 경우에만 스태킹을 허용한다: (1) 긴급도 High 신호가 복수 동시 발생, (2) 각 카드가 서로 다른 태스크 도메인. 스태킹 허용 시 한 세션의 홈 화면에 동시 노출 가능한 Micro Task는 최대 3개입니다. 4개 이상 산출된 경우 긴급도 순으로 정렬 후 상위 3개만 렌더링합니다.
R2노출 순서: High 긴급도 → Medium 긴급도 → 동일 긴급도 내에서는 CVM 전환 가치 높은 순.
R3Micro Task가 0개인 경우 해당 슬롯을 빈 상태로 두지 않습니다. ISR Fallback 규칙에 따라 일반 개인화 콘텐츠 카드로 대체합니다.
R4고객이 동일 Micro Task를 2회 연속 거절(닫기)하면 해당 세션에서 해당 태스크를 재노출하지 않습니다.
R5Micro Task 완료(CTA 실행) 시 해당 카드는 즉시 제거되고, 다음 순위 Micro Task가 있으면 자연스럽게 채웁니다.

Rules
