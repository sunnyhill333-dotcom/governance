# 거버넌스 연결접점 참조 가이드
> 정책서 → SB → 목업 프로세스에서 각 거버넌스 문서가 어느 단계에, 어떤 역할로 개입하는지 정리한 참조 문서입니다.

---

## 전체 흐름

```
정책서 (Hard Lock)
    ↓ [UXP 참조]
SB 생성
    ↓ [UXPT · VOT · SDG 참조]
목업 생성
```

---

## 파일별 연결접점

| 문서 | 파일 | 개입 단계 | 읽는 대상 | 결정하는 것 |
|---|---|---|---|---|
| **정책서** | `usecase_US-*.md` | 전 단계 기준 | 유즈케이스 / 프로세스 / 기능 / Policy Item | 화면 수 · 플로우 · 기능 입출력 · 예외 처리 · 텍스트 정책 |
| **UXP** | `UXL_ETR/EXP/SCH/DCD/ACT/TCP/RSV` | SB 생성 | 기능(처리 흐름) + 여정 구분 | 오가니즘 배치 순서 · 섹션 노출 우선순위 · 정보 위계 · 조건부 노출 기준 |
| **UXPT** | `UX_Pattern_1/2.md` | 목업 생성 | 기능(실패·예외 케이스) | 로딩 방식 · 에러 표시 · 리커버리 경로 · 버튼 계층 · 화면 전환 방식 |
| **VOT** | 보이스톤 가이드 | 목업 생성 | Policy Item (텍스트·안내 규칙) | 버튼 라벨 · 안내 문구 · 에러 메시지 · 추천 카피 · 관계 수준별 어조 |
| **SDG** | `SDG_*.md` | 목업 생성 | SB 오가니즘 ID · 서버제어항목 | 오가니즘 ID 네이밍 · 서버 payload 구조 · 페이지 단위 조합 방식 |

---

## 검수 체크포인트

### SB 검수
- [ ] 정책서 프로세스 단계와 화면 수 일치
- [ ] 오가니즘 누락 없음
- [ ] 조건부 섹션 노출 기준 정합
- [ ] UXP 여정 원칙 반영 (DO 원칙 적용 여부)
- [ ] 섹션 우선순위 일치

### 목업 검수
- [ ] 기능(실패·예외 케이스) 처리 오가니즘 포함
- [ ] UXPT 상태처리 패턴 적용 (로딩·에러·리커버리)
- [ ] VOT 카피 톤 준수
- [ ] SDG 오가니즘 ID 네이밍 기준 충족
- [ ] 서버 payload 구조 정합

## 충돌 발생 시 원칙

정책서는 기본값이다. 충돌이 발생했을 때 판단이 서지 않으면 정책서를 따른다. 단, 정책서를 따른 결과가 사용자에게 실질적 혼란·이탈·오류를 유발한다고 판단될 경우, 거버넌스는 정책서 수정을 제안할 권한을 가진다. 어느 방향으로 결론이 나든, **충돌을 묵인하거나 기록 없이 넘어가는 것은 항상 금지**다. 모든 충돌은 반드시 검수 리포트에 남기고 의사결정 이력을 보관한다.

### 충돌 유형 판단

**Type A — 거버넌스가 적응** : 정책서 구조는 유효하나 거버넌스를 100% 구현하기 어려운 경우. 정책 범위 안에서 최대한 적용하고 사유를 검수 리포트에 기록한다.

**Type B — 정책서 수정 제안** : 거버넌스 원칙을 지키지 못할 경우 사용자 경험이 임계치 이하로 떨어지는 경우. 아래 항목을 포함한 수정 제안서를 작성해 의사결정자에게 전달한다.
- 충돌 지점
- 거버넌스 원칙이 지켜져야 하는 이유 (사용자 영향 근거)
- 정책 수정 제안 내용
- 미수정 시 예상 리스크
